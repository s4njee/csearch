# CSearch

CSearch is a monorepo for ingesting, storing, querying, and presenting U.S. congressional bill and vote data.

```text
GovInfo + congress.gov
        |
        v
backend/scraper  ->  PostgreSQL  ->  backend/api  ->  frontend
                          |               ^
                          |    Redis      |
                          +---(cache)-----+
```

## Repository Map

| Path | What lives here |
| --- | --- |
| `backend/scraper/` | Go ingest pipeline and vendored Python `congress` scraper. Owns schema bootstrap, parsing, hash-based skip logic, and Redis cache invalidation. |
| `backend/api/` | Fastify API with Knex queries, Redis-backed route caching, and structured Pino logging. |
| `frontend/` | Nuxt 4 app with static S3/CloudFront publishing, nginx container deploys, and local dev mode. |
| `argo/` | Argo CD `Application` manifests — the top-level deployment control plane. |
| `k8s/` | Kubernetes workload manifests synced by Argo applications. |
| `k8s/logging/` | Fluent Bit config, DaemonSet, tiny collector, and optional Grafana dashboards. |
| `docs/` | Engineering onboarding and operational docs. |
| `archiver/` | Utility container for archiving the congress data directory. |

## Components at a Glance

**Scraper** — Runs as a Kubernetes CronJob (daily at midnight CT). Fetches bill and vote source files via the vendored Python scraper, parses only changed files using SHA-256 hash caches, and upserts normalized data into Postgres. Covers bills from the 93rd Congress and votes from the 101st Congress through current.

**Database** — Postgres is the system of record. Schema bootstrap comes from `backend/scraper/schema.sql`.

**API** — Fastify service reading from Postgres. Serves bill lists and detail, vote lists and detail, search, member and committee pages, and analytical explore queries. Hot routes are cached in Redis with a 24-hour TTL.

**Frontend** — Nuxt 4 static site deployed to S3 + CloudFront for the public site at csearch.org. Also supports nginx container deployments for cluster-hosted environments like test.csearch.org.

**Logging** — API and scraper write structured JSON to stdout. Fluent Bit tails container logs and ships them to the in-cluster collector or directly to S3.

## Quick Start

```bash
# API
cd backend/api && npm install && npm test
POSTGRESURI=localhost DB_PORT=5433 REDIS_URL=redis://localhost:6379 npm run dev

# Frontend
cd frontend && npm install
NUXT_API_SERVER=http://localhost:3000 npx nuxt dev

# Scraper
cd backend/scraper && go test ./...
```

## Important Conventions

- Do not hand-edit `backend/scraper/csearch/*.go` — those files are generated by `sqlc`.
- `backend/scraper/explore.sql` is the source of truth for explore queries; `backend/api/sql/explore.sql` is a copied build artifact.
- `backend/scraper/congress/` is vendored upstream code.
- `argo/applications/` is the default deployment entry point.
